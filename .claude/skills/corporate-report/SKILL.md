# 企業分析レポート生成

証券コード `$0`、企業名 `$1` の企業分析レポートを作成する。

## 作業手順

### 1. プロジェクト準備

- `{証券コード}_{企業名英語小文字}/` ディレクトリを作成（例: `5819_canare/`）
- その中に以下のファイルを作成:
  - `report.md` — 企業分析レポート本体
  - `charts.html` — Chart.jsベースの財務データグラフ
  - `README.md` — 企業トップページ（企業一覧への戻りリンク付き）
- ルートの `README.md` の企業一覧テーブルに新しい企業を追加

### 2. 情報収集

以下の情報を有価証券報告書・決算短信・IR資料から収集する:

- **会社概要**: 社名、証券コード、本社、代表者、設立、決算月、従業員数、時価総額
- **沿革**: 創業〜現在の主要イベント
- **事業内容**: 製品構成（売上・構成比・前期比）、セグメント構成
- **市場・競合**: 市場規模、競合他社、事業構造の特徴
- **業績推移**: 連結経営指標（5期以上）、連結PL、四半期推移、セグメント別業績
- **財務諸表**: BS詳細、PL詳細、CF計算書（10年推移推奨）
- **株主動向**: 大株主リスト、配当推移、資本政策方針、自社株買い実績
- **ガバナンス**: 取締役構成、監査法人、KAM、関連当事者取引
- **リスク**: 有報記載のリスク、特記事項

### 3. レポート構成（report.md）

```markdown
# {企業名}（{証券コード}）企業分析レポート

## 会社概要
## 沿革
## 事業内容
### 製品構成
### セグメント構成
## 市場規模・競合
## 業績推移
### 連結経営指標等の推移
### ROE vs 事業ROIC（資本効率の真実）
  - 定義: 事業ROIC = NOPAT（税引後営業利益）÷ 事業投下資本
  - NOPAT = 営業利益 ×（1 - 実効税率）
  - 事業投下資本 = 純資産 - ネットキャッシュ
### 連結損益計算書
### 四半期推移
### セグメント別業績
## 要約財務諸表
### 連結BS推移
### 連結BS詳細
### 連結PL詳細
### 連結CF計算書（10年推移）
### 運転資本効率
## 株主動向・資本政策
### 大株主リスト
### 配当推移
### 資本政策の基本方針
### コーポレートガバナンス
## 業績予想
## 企業価値
### 資産バリューチェック（清算価値）
### 収益バリューチェック（DCF法）
  - 弱気: NC + FCF/R（成長なし）
  - ミドル: 5年10%成長（最も現実的）
  - 強気: 5年20%成長（楽観前提を明記）
  - シナリオ比較表
## 投資判断
### 五段階評価（A〜E）
### 総括
### 株価が評価されにくい構造的背景（ベアケース）

  - カタリストの有無と実現可能性

## 事業等のリスク
## 株価指標
```

### 4. カラーハイライト

レポートに以下のCSSクラスでハイライトを付ける:

```html
<style>
.hl-red { background: #fdecea; color: #c0392b; font-weight: 600; padding: 0.1em 0.4em; border-radius: 3px; }
.hl-green { background: #eafaf1; color: #1e8449; font-weight: 600; padding: 0.1em 0.4em; border-radius: 3px; }
.hl-amber { background: #fef9e7; color: #b7950b; font-weight: 600; padding: 0.1em 0.4em; border-radius: 3px; }
</style>
```

- **赤**: 割安シグナル・要注目リスク（PBR<1、EV<0、急落した指標など）
- **黄**: 改善余地あり・注意（ROE低水準、予想減益など）
- **緑**: ポジティブ（成長セグメント、高評価項目、財務健全性など）

### 5. チャート構成（charts.html）

Chart.js v4 + chartjs-plugin-annotation を使用。推奨グラフ:

1. 売上高・利益の推移（棒+折れ線の複合）
2. セグメント別 売上高・利益
3. 四半期 営業利益率の推移（季節性パターンをポイント色で表現）
4. BPS vs 株価の推移
5. ネットキャッシュ vs 時価総額
6. 営業CF・実質FCFの推移
7. ROE vs 事業ROIC の推移（8%ラインの注釈付き）
8. 海外売上比率の推移（該当する場合）
9. 配当の推移

各グラフに:
- 左ボーダーのカラーハイライト（hl-red / hl-amber / hl-green）
- annotationプラグインで参考ライン・ゾーン
- 注目データポイントのポイント色変更
- 注釈テキストに背景色

### 6. 分析上の注意点

- **ROIC**: 必ずNOPAT（税引後営業利益）基準で計算する。純利益ベースは不正確
- **DCF**: 強気ケースの前提が実績CAGRから乖離する場合は明記する
- **ベアケース**: 事実ベースで構造的課題を整理する。攻撃的にならず、応援目線で書く
- **ブルケース**: 希望ではなく実績データで裏付ける
- **棚卸資産評価損など重要リスク**: リスクセクションだけでなく総括にも言及する
- **推計値**: 推計を含む場合は必ず注記する（*マーク等）

### 7. Jekyll / GitHub Pages 対応

- report.md の先頭にJekyll front matterは不要（READMEから.mdリンク、Jekyll側で.html化）
- charts.htmlにはJekyll front matter `layout: default` を付ける
- ファイル名・ディレクトリ名は英語にする（URLエンコード問題回避）
- ナビリンクに「企業一覧（../）」への戻りリンクを含める
