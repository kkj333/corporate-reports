# レポート更新

`$ARGUMENTS` で指定された企業ディレクトリの report.md と charts.html を、最新の決算データで更新する。

---

## 手順

### 1. 現状把握

- 既存の `report.md` を読み、現在どの期までのデータが入っているか確認する
- `data/` フォルダに新しい抽出データがあるか確認する（なければユーザーに確認）
- `charts.html` のデータ配列の最新期を確認する

### 2. 更新対象の特定

以下を新しいデータで更新する:

**report.md:**
- 冒頭の株価・作成日
- 連結経営指標テーブル（新しい期の列を追加）
- ROE vs 事業ROIC テーブル（新しい期を追加、NOPAT再計算）
- 連結PL・BS・CF（最新2期に差し替え）
- 四半期推移（新しい四半期を追加）
- セグメント別業績（最新期に更新）
- 配当推移（新しい期を追加）
- 業績予想（新しい会社予想に更新、過去予想の精度を追記）
- 企業価値（清算価値・DCFを最新BSとFCFで再計算）
- 株価指標（最新株価で再計算）
- 総括・ベアケース・ブルケース（必要に応じて更新）

**charts.html:**
- 各グラフのデータ配列に新しい期のデータを追加
- ラベル配列にも新しい期を追加
- Y軸の最大値を必要に応じて調整

### 3. 更新ルール

- 既存のカラーハイライトは維持する。新しいデータにも適切にハイライトを付ける
- ROICは必ずNOPAT基準で計算する
- DCFのFCFは最新期の実績に更新する
- 会社予想の保守性テーブルに、前期予想 vs 実績の行を追加する
- 変更箇所が多い場合はセクション単位で更新し、差分を確認しながら進める

### 4. 更新後チェック

- 数値の整合性（PL合計、BS貸借一致、CF残高等）
- ハイライトが適切か（新しい要注目データに色が付いているか）
- グラフのデータ数とラベル数が一致しているか
- ナビリンクが壊れていないか
- 更新日を冒頭に反映する
