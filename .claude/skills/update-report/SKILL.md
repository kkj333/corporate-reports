# レポート更新

`$ARGUMENTS` で指定された企業ディレクトリの report.md を、最新の決算データで更新する。

---

## 手順

### 1. 現状把握

- 既存の `report.md` を読み、現在どの期までのデータが入っているか確認する
- `data/` フォルダに新しいデータがあるか確認する
- `chart_config.json` のデータ配列の最新期を確認する

### 2. データソースの確認と取得

以下の優先順位でデータを使用する:

| 優先度 | データソース | 取得方法 | 特徴 |
|---|---|---|---|
| 1 | EDINET CSV（有報） | `/download-edinet` で自動取得 | 数値が正確、構造化済み。ただし有報は決算から3ヶ月後に提出されるため期が古い |
| 2 | 決算短信 PDF | 企業IRページから手動で `data/pdf/` に配置 | 最新の決算データ。EDINETでは取得不可（TDnet管轄） |
| 3 | 有報 PDF | `/download-edinet` で自動取得 | CSVが利用できない場合（2024年3月以前の書類）に使用 |

**注意:**
- 決算短信はEDINET APIでは取得できない。企業IRページ等からPDFを手動ダウンロードし `data/pdf/tanshin_YYYYQN.pdf` として配置する
- `data/` にデータがない場合はユーザーに確認する
- EDINET CSVと既存レポートが同じ期のデータの場合、突合のみ行い更新不要と判断する

### 3. 更新対象の特定

既存レポートより新しい期のデータがある場合、以下を更新する:

**report.md:**
- 冒頭の株価・作成日
- 連結経営指標テーブル（新しい期の列を追加）
- ROE vs 事業ROIC テーブル（新しい期を追加、NOPAT再計算）
- 連結PL・BS・CF（最新2期に差し替え）
- 四半期推移（新しい四半期を追加）
- セグメント別業績（最新期に更新）
- 配当推移（新しい期を追加）
- 業績予想（新しい会社予想に更新、過去予想の精度を追記）
- 企業価値（清算価値・DCFを最新BSとFCFで再計算）
- 株価指標（最新株価で再計算）
- 総括・ベアケース・ブルケース（必要に応じて更新）

**chart_config.json:**
- 各チャートの echarts_option 内のデータ配列に新しい期のデータを追加
- xAxis の data 配列にも新しい期のラベルを追加
- 更新後 `uv run corporate-reports build-report $ARGUMENTS` で report.html を再生成

### 4. 更新ルール

- ROICは必ずNOPAT基準で計算する
- DCFのFCFは最新期の実績に更新する
- 会社予想の保守性テーブルに、前期予想 vs 実績の行を追加する
- 変更箇所が多い場合はセクション単位で更新し、差分を確認しながら進める

### 5. 更新後チェック

- 数値の整合性（PL合計、BS貸借一致、CF残高等）
- chart_config.json のデータ数とラベル数が一致しているか
- ナビリンクが壊れていないか
- 更新日を冒頭に反映する
